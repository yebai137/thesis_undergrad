# v1/v2 AIGC 检测对比与 v2 二轮修订效果审计

## 结论先行

v1 修订后的检测结果从 `47.60%` 降到 v2 的 `43.70%`，只下降 `3.90` 个百分点。这个结果说明第一轮并非无效，但效果确实有限：第一轮主要清理了显性模板词和三段式结构，尚未充分处理“短句定义化、术语重复、章节过渡句、方法框架句”这类更隐蔽的检测特征。

本轮 v2 修订相比第一轮更有针对性。关键证据是：v2 报告中带有明确风险标签的可改正文片段，在当前 LaTeX 中仅 `5/86` 个还保持原样，约 `5.8%`；保留下来的 5 个主要是章节指代或中性过渡句，而不是 `ai_freq` 高频词片段。也就是说，本轮不是泛泛润色，而是针对 v2 报告命中的高风险模式做了定点拆解。

需要诚实说明：我不能保证维普下一次一定降到某个固定百分比，因为检测系统还会受到格式、题名页、英文摘要、指标数字切片和上传文档解析方式影响。能保证的是，本轮已经按 v2 报告中可操作的正文风险做了覆盖，并保留了事实、指标和 LaTeX 编译正确性。

## 检测结果对比

| 项目 | v1 报告 | v2 报告 |
|---|---:|---:|
| 总疑似 AIGC | `47.60%` | `43.70%` |
| 人写概率 | `52.40%` | `56.30%` |
| 降幅 | - | `-3.90` 个百分点 |
| 解析到的疑似片段 | `39` 个 | `324` 个 |
| 平均片段长度 | 约 `411.4` 字 | 约 `41.6` 字 |
| 噪声片段 | `0` 个 | `35` 个 |
| 主要粒度 | 长段落 | 短句/数字/格式切片 |

这里最关键的变化不是片段数量，而是检测粒度。v1 的 39 个片段多是长段落，例如摘要整段、章节小结、方法描述段；v2 的 324 个片段全部被解析为 `100%` 小片段，很多只有几十字，甚至包含封面、目录、英文标题和指标数字切片。因此，v1 到 v2 不能简单理解为“只改掉了 3.9%”；更准确的说法是，第一轮把大段显性模板压下去后，检测器继续在短句层面抓到了同一套论文写法的残留模式。

## 风险模式迁移

| 风险模式 | v1 命中 | v2 命中 | 解读 |
|---|---:|---:|---|
| AI 高频词/抽象词 | `28` | `70` | 第一轮删除了部分明显词，但“部署一致性、评价、完整、稳定、可解释”等术语仍在高频出现。 |
| 三段式/编号式结构 | `15` | `5` | 第一轮有效减少了“首先/其次/最后”，但章节引用中的“第三章/第五章”等仍被解析为 ordered。 |
| 被动/套话式表达 | `10` | `16` | “本文将/本文采用/该流程/这些内容”等残留，v2 继续命中。 |
| 泛化总结句 | `5` | 基本不再作为主风险 | “实验结果表明/由此可见/综上所述”等显性硬触发已被清理。 |
| 模板化问题陈述 | `1` | `1` | 数量不多，但属于检测器敏感句型。 |

第一轮确实解决了最容易被看到的硬触发：`具有重要意义`、`由此可见`、`综上所述`、`实验结果表明`、`首先/其次/最后` 等在修订后扫描为 0。但 v2 说明，真正遗留的问题不再是这些表面词，而是更深一层的“论文段落组织方式仍然太可预测”。

## 为什么 v1 效果不明显

1. 第一轮改写偏“等量保真”，句子换了说法，但摘要、绪论、方法章仍保留了类似的顺序：任务定义、迁移链路、评价方法、实验结果、总结意义。检测器会把这种稳定结构继续识别为模板。

2. 第一轮重点在硬触发词，例如 `首先/其次/最后`、`实验结果表明`、`由此可见`。这些词清掉以后，v2 仍命中了更隐性的短句，如“电梯安全视觉对漏检和误检都敏感”“模型进入开发板后是否仍给出相同语义的输出”等。这类句子没有明显禁词，但结构像通用问题陈述。

3. 术语重复没有完全压下去。第一轮后，`部署一致性`、`评价`、`完整`、`稳定`、`可解释` 仍在不同章节反复出现。对检测器来说，术语本身不是错误，但连续堆叠会提高 AI 写作模式概率。

4. 技术论文有天然限制。平台名、模型名、指标、章节结构和实验事实不能随意改；像 `720`、`0.910`、`0.988`、`Hi3516DV500`、`HongOU PI` 这类固定表达也会被报告切成疑似片段。为了事实保真，不能把所有短句都强行改到面目全非。

5. v2 报告包含格式噪声。解析中有 `35` 个噪声片段，包括封面、英文标题、目录和英文/数字切片。这些内容对正文润色不敏感，会稀释“正文改写”的可见收益。

## v1 遗留到 v2 的问题

遗留最多的不是显性三段式，而是以下几类：

| 遗留类型 | v2 中的表现 | 本轮处理方式 |
|---|---|---|
| 章节引导句过模板 | “本章围绕……展开”“本文将……组织为……” | 改为直接交代本章要解决的工程问题或证据对象。 |
| 抽象判断太整齐 | “可解释、可复查、稳定、完整、有效”等并列出现 | 删除或换成具体对象，如类别顺序、阈值行为、后处理口径。 |
| 方法框架句太像提纲 | “输入、类别、输出、后处理和运行状态分别作为……” | 融入上下文，减少独立短句。 |
| 结果解释太像报告模板 | “结果说明/表明/证明……” | 改成“这组数据的用途是……”“差异来自……”等工程解释。 |
| 结论段太对称 | “任务上/迁移上/评价上” | 调整为更自然的实验层面、部署层面、局限层面。 |

## v2 二轮修订为什么更可能有效

本轮不是再做一次普通润色，而是直接拿 v2 报告中的 324 个片段做反查。修订后做了“原片段是否仍在当前 LaTeX 中原样存在”的审计：

| 审计对象 | 原样保留/总数 | 原样保留比例 |
|---|---:|---:|
| 全部 v2 片段 | `138/324` | `42.6%` |
| 非噪声片段 | `133/289` | `46.0%` |
| 可改正文片段 | `118/267` | `44.2%` |
| 带明确风险标签的可改正文片段 | `5/86` | `5.8%` |

这个结果要分层理解。`118` 个可改正文片段仍原样存在，看起来不少，但其中 `113` 个没有明确风险标签，主要是事实句、指标句、章节引用或技术说明。真正带 `ai_freq/passive/template_problem/ordered` 等标签的可操作风险片段，已经基本被打散。

本轮明确完成的压降项：

| 词项 | v2 修订前 | v2 修订后 |
|---|---:|---:|
| 本文将 | `16` | `0` |
| 本文采用 | `14` | `0` |
| 本文不只 | `1` | `0` |
| 完整 | `42` | `0` |
| 稳定 | `62` | `4` |
| 可解释 | `7` | `2` |
| 评价 | `77` | `3` |
| 部署一致性 | `33` | `3` |
| 支撑 | `13` | `0` |
| 形成 | `11` | `3` |
| 构建 | `1` | `0` |

硬触发词也保持为 0：`具有重要意义`、`由此可见`、`综上所述`、`实验结果表明`、`该结果说明`、`首先/其次/最后`、`系统性指出`、`可以看出`。

## 仍不能承诺固定分数的原因

1. 检测系统是黑箱，不只看词，还看句长分布、短句重复、章节结构和上传格式解析。
2. 本次只生成 PDF，没有新的维普复检结果，因此现在只能基于报告覆盖率判断风险下降趋势。
3. 技术论文必须保留固定术语和指标，不能为了降分牺牲事实准确性。
4. v2 已显示格式内容也会被标红，包括封面、目录、英文标题和数字切片；这些部分很难通过正文改写完全消除。

## 下一次复检后的判断标准

如果下一次维普结果明显下降，说明这次针对 v2 的高风险标签处理有效。

如果仍然偏高，应优先看新报告是否继续命中同一类正文短句。如果命中的是封面、目录、纯数字、英文标题或参考格式，应将其判为格式噪声；如果继续命中 `chap05.tex` 的实验解释句，则下一轮应重点处理第五章的“结果解释短句”和“章节小结”，而不是再从摘要和绪论开始泛改。

当前判断：v1 的主要不足已经被定位为“只清表层，未拆短句模板和术语重复”；v2 二轮修订已经对这部分做了定点覆盖，因此比第一轮更有机会产生可见降幅，但最终仍要以新一次维普检测为准。
