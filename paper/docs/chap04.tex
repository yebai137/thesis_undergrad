\chapter{数据集、场景复杂性与评测协议}
\label{cha:data_protocol}

\section{任务与数据组织}

本文实验围绕电梯轿厢内 \textit{person + ebike} 双目标检测展开。图像验证集用于评价检测精度，代表性视频片段用于观察连续场景稳定性，板端运行输出用于分析模型迁移后的阈值行为、运行状态和分段耗时。难例不作为独立分类实验，也不额外建立人工分组；本文只在讨论中使用场景复杂性因素解释模型在真实轿厢中为什么会出现漏检、误检或边框漂移。

为了使实验结论服务于第一章提出的问题链，本文将评价目标拆分为六个具体问题：第一，模型迁移到板端后是否能完整处理验证集并保持双目标检测能力；第二，\textit{person} 与 \textit{ebike} 的分类别指标是否符合任务语义；第三，置信度阈值变化是否会显著改变安全输出；第四，NMS 与应用层后处理是否影响最终候选框；第五，代表性视频片段中输出是否保持连续稳定；第六，batch 图像验证耗时、单帧核心处理耗时和模型执行阶段耗时是否能够被明确区分。第五章的结果组织即围绕这些问题展开。

\section{场景复杂性因素}

电梯场景的困难之处来自封闭空间、固定视角和安全响应要求的叠加。轿厢内常见的高亮区域、门框结构、局部目标、近距离重叠、尺度变化和暗部区域都会改变目标轮廓、类别分数或候选框排序。本文不把这些因素整理成人工标签，也不按人工分组报告独立指标；它们只用于解释阈值、NMS、后处理和视频稳定性实验中的现象。

本文使用的证据可以分为四类。第一类是 720 张图像验证集，用于统计总体精度、分类别精度和板端 batch 验证耗时；第二类是基于既有板端检测输出的阈值后验重算，用于观察同一批输出在不同安全阈值下的稳定性；第三类是 batch 图像复查和连续视频处理中的分段计时输出，用于区分 batch 端到端耗时、单帧核心处理和模型执行阶段；第四类是代表性视频片段，用于观察连续帧中的漏检、闪烁和边框抖动。上述证据共同回答本文核心问题：同一 YOLO 模型迁移到 OM 和板端运行后，是否仍保持可解释、可复查的检测行为。

上述场景因素不要求额外训练识别模型，也不改变本文的评价单位。这样可以避免论文主线发散，同时保留电梯场景相对于通用检测任务的特殊性。

\section{评测指标设计}

本文评测指标分为检测精度、部署一致性和运行时性能三类。检测精度包括 precision、recall、F1 和 mAP@0.5，用于描述模型在验证集上的基础识别能力。部署一致性主要通过分类别统计、阈值敏感性和运行状态进行分析；跨阶段逐样本边框漂移属于更完整的一致性实验设计，在本文中作为后续扩展。运行时性能包括 batch 图像验证链路端到端单图处理耗时、成功处理样本数和分段计时，用于判断系统运行结果是否可复查。

具体指标围绕三个问题组织。检测能力由 precision、recall、F1 和 mAP@0.5 描述；类别语义由 person 与 ebike 的 TP、FP、FN、recall 和 mAP@0.5 检查；部署行为由多阈值 F1、类别召回、成功样本数、端到端耗时和分段计时共同说明。跨阶段 IoU 漂移属于更完整的一致性实验设计，本文只在方法层面保留其评价方式。

\section{实验协议}

为保证实验结果可比较，本文采用统一协议。首先，所有图像级指标均在同一验证集上统计，并统一保留三位小数。其次，阈值敏感性实验只改变置信度阈值，其余参数保持不变。再次，分段计时实验必须明确运行模式，区分 batch 图像验证链路和连续视频链路。最后，视频场景分析只使用连续场景片段表述，不在正文中展开数据文件位置，也不在缺少逐帧人工标注的条件下计算视频级 precision 或 recall。

各实验只改变一个主要变量。基线验证固定验证集、类别顺序和输入尺寸；阈值敏感性只改变置信度阈值；NMS 敏感性只改变 IoU 阈值；后处理消融只改变电动车清理策略；视频稳定性只改变平滑窗口；分段计时则固定模型、二进制和阈值设置，只比较 batch 端到端、单帧核心处理和模型执行阶段。该协议体现了本文的研究重点：实验不追求数量堆叠，而是围绕 YOLO 模型的边缘部署一致性建立可复查的评价流程。

\section{场景复杂性与部署误差的关系}

场景复杂性与部署一致性之间存在直接联系。高亮区域和暗部区域主要影响类别置信度，容易在阈值扫描中表现为召回率波动；轿厢边界和门框结构主要影响边框回归和 NMS，容易在边框解释评价中表现为 IoU 下降；近距离重叠则同时影响类别排序和候选框抑制，可能造成不同阶段输出数量不一致。

因此，本文在分析实验结果时采用“指标 + 场景解释”的方式。若 mAP@0.5 下降，需要进一步判断下降来自类别分数变化还是边框定位变化；若视频片段出现闪烁，需要结合阈值敏感性判断是否存在置信度边界问题。通过这种方式，场景复杂性服务于部署一致性分析，而不是另起一条难例创新主线。


\section{数据集划分与可复查性要求}

为保证实验结果能够被复查，数据划分需要保持清晰。本文使用的图像验证集固定为 720 张图像，用于报告板端总体指标和分类别指标。该验证集在正文中只描述其规模、类别和用途，不展开本地文件组织细节。

数据可复查性包括三层含义。第一，样本数量可复查，即实验表格中的样本数与指标来源一致。第二，类别统计可复查，即 person 与 ebike 的真实目标数、预测数、TP、FP、FN 能够对应到评测结果。第三，运行状态可复查，即成功样本数、失败样本数和端到端耗时能够说明板端验证是否完整执行。

对于视频片段，在缺少人工逐帧标注的条件下，本文不计算视频级 precision 或 recall，而只使用机器侧统计描述输出稳定性。这样可以保证评价指标与可用证据相匹配。


\section{跨阶段一致性实验设计}

跨阶段一致性实验的理想形式是选取同一批输入样本，分别在训练端、ONNX 端、OM 端和板端运行，再对输出结果进行逐样本匹配。匹配时首先按照类别和 IoU 阈值确定同一目标，然后统计类别一致率、平均边框 IoU、置信度差异和最终检测数量差异。该实验能够直接衡量模型迁移过程是否改变检测行为。

在实际执行时，应避免同时改变多个变量。输入图像、预处理方式、置信度阈值和 NMS 阈值需要固定，唯一变化的是推理阶段。若发现输出差异，再根据第三章的误差定位表检查类别契约、坐标契约或后处理契约。这样，跨阶段实验不仅给出结果，还能帮助定位问题来源。

本文已经形成完整板端验证结果，但训练端、ONNX 端、OM 端和板端的逐样本统一对比仍属于研究局限。因此，本节仅给出跨阶段一致性实验的评价方式，当前实证结果仍以板端完整验证、阈值后验重算和分段计时为主。


\section{实验结果呈现原则}

为了保证实验结果可信，本文区分实测结果、后验重算结果和方法性评价设计。已有完整证据支撑的内容，如 720 张图像的板端验证结果，可以写入实验结果表并进行结论分析；基于既有检测输出重新筛选得到的阈值敏感性结果，可以作为补充实验；跨阶段逐样本对齐尚不具备完整数值表，因此只用于说明评价方法和研究局限。

这一处理符合实验研究的基本要求。工程实践中存在日志、脚本、视频和人工观察等多种材料，只有区分证据强度，才能避免把经验观察混同为定量结论。本文将图像验证集用于计算检测指标，将视频片段用于辅助观察稳定性，将场景复杂性描述用于解释误差来源。

通过这种方式，本文的实验结论只建立在可复查证据之上，跨阶段对齐则作为研究局限与扩展方向处理。

\section{本章小结}

本章给出了本文的数据组织方式、场景风险分类、评测指标体系和实验协议。场景风险分类服务于误差解释和实验分组，评价重点则集中在 YOLO 双目标检测模型在海思/昇腾类边缘开发板上的部署一致性。
