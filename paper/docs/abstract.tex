\cabstract{
电梯轿厢是典型的封闭式高频公共微空间，具有反光强、遮挡多、视角近、目标尺度变化快和事件响应时延要求高等特点。针对电动车入梯风险治理中离线检测精度与边缘部署表现可能不一致的问题，本文以电梯轿厢场景中的行人与电动车双目标检测为研究对象，围绕 YOLO 双目标检测、海思/昇腾类边缘开发板兼容迁移和部署一致性验证开展研究。本文的核心思路是：电梯边缘安全视觉的可用性不仅取决于模型能否识别目标，还取决于任务建模是否贴合风险、迁移接口是否保持语义一致、实验指标是否能够解释板端真实运行口径。

首先，本文分析电梯轿厢的空间结构、成像特点和安全响应需求，明确行人与电动车双目标检测任务边界。其次，结合 YOLO 检测器的 Backbone、Neck、Head 和后处理流程，将自训练权重到 ONNX 中间表示、OM 离线模型和板端推理的过程抽象为输入、类别、输出、后处理和运行五类迁移接口契约。最后，针对模型迁移到边缘设备后可能出现的类别语义、边框解释、阈值行为和运行时口径变化，构建部署一致性评价框架，并以完整验证集、分类别指标、阈值后验重算、分段计时和代表性视频片段作为证据来源。

实验结果表明，在 720 张图像构成的验证集上，边缘设备端总体 precision 为 0.901，recall 为 0.923，F1 为 0.912，mAP@0.5 为 0.910；其中行人类别 mAP@0.5 为 0.869，电动车类别 mAP@0.5 为 0.952。阈值敏感性后验重算显示，在 0.20--0.35 置信度阈值范围内，总体 F1 保持在 0.915--0.922 区间，电动车召回率保持在 0.956 以上。板端 batch 图像验证链路端到端平均单图处理耗时约为 1120 ms；进一步的分段计时表明，50 张图像 batch 复查中模型执行阶段均值约为 8.2 ms，代表性连续视频处理中模型执行阶段均值约为 8.4 ms。上述结果说明，本文方法能够在电梯边缘视觉场景中形成较稳定的双目标检测能力，并提供可复查的部署一致性证据。
}

\ckeywords{电梯安全视觉；边缘智能；目标检测；部署一致性；电动车入梯检测}

\eabstract{
Elevator cabins are closed, high-frequency public micro-spaces characterized by strong reflections, frequent occlusions, close viewpoints, rapid scale variation, and strict latency requirements for safety response. To address the potential gap between offline detection accuracy and edge-device behavior in e-bike-in-elevator risk governance, this thesis studies pedestrian and e-bike detection in elevator cabins, focusing on YOLO-based dual-target detection, compatible migration to HiSilicon/Ascend-like edge boards, and deployment consistency verification. The central idea is that the usability of elevator edge safety vision depends not only on whether a model detects targets, but also on whether the task formulation matches the safety risk, whether the migration interface preserves semantics, and whether the evaluation metrics explain the actual board-side runtime behavior.

First, this thesis analyzes the spatial structure, imaging characteristics, and safety requirements of elevator cabins, and defines the dual-target detection task for pedestrians and e-bikes. Second, based on the Backbone, Neck, Head, and post-processing pipeline of YOLO detectors, the migration from self-trained weights to ONNX, OM offline models, and board-side inference is formulated as five interface contracts: input, category, output, post-processing, and runtime contracts. Finally, to analyze behavior changes after edge deployment, a deployment consistency framework covering category semantics, bounding-box interpretation, threshold behavior, and runtime measurements is constructed, using full validation, per-class metrics, posterior threshold recomputation, segmented timing, and representative video snippets as evidence sources.

Experimental results show that on a 720-image validation set evaluated on the edge device, the system achieves an overall precision of 0.901, recall of 0.923, F1 of 0.912, and mAP@0.5 of 0.910. The pedestrian class reaches 0.869 mAP@0.5, while the e-bike class reaches 0.952 mAP@0.5. A posterior threshold-sensitivity recomputation shows that the overall F1 remains within 0.915--0.922 under confidence thresholds from 0.20 to 0.35, and the e-bike recall remains above 0.956. The end-to-end average processing time of the board-side batch image validation path is approximately 1120 ms per image; additional segmented timing shows that the mean model-execution stage is about 8.2 ms in a 50-image batch check and about 8.4 ms in a representative continuous-video run. These results demonstrate that the proposed approach provides stable dual-target detection capability and reproducible deployment-consistency evidence for elevator edge-vision scenarios.
}
