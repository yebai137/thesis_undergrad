\chapter{结论与展望}
\label{cha:conclusion}

本文面向电梯轿厢场景中的电动车入梯风险，研究了基于 YOLO 的行人与电动车双目标检测模型及其在海思/昇腾类边缘开发板上的兼容迁移问题。与只关注离线检测精度的研究不同，本文将训练权重、ONNX 中间表示、OM 离线模型、板端推理和后处理输出纳入同一评价框架，重点分析模型迁移后类别语义、边框解释、阈值行为和运行状态是否具备可复查的一致性。

本文的主要工作可以概括为三点。第一，面向电梯轿厢建立了 \textit{person + ebike} 双目标检测任务，并结合反光遮挡、门口结构、电动车局部可见、拥挤重叠等场景因素解释检测误差来源。第二，提出了自训练 YOLO 模型到海思/昇腾类边缘开发板的兼容迁移方法，围绕输入预处理、类别顺序、输出张量、后处理参数和运行状态建立接口契约。第三，构建了面向安全视觉的部署一致性评价思路，通过板端完整验证集结果、分类别指标、阈值敏感性后验重算、分段计时和代表性视频统计分析迁移后的实际行为。

实验结果表明，在 720 张图像验证集上，边缘设备端总体 precision 为 0.901，recall 为 0.923，F1 为 0.912，mAP@0.5 为 0.910，板端 batch 图像验证链路端到端平均单图处理耗时约为 1119.510 ms。其中，行人类别 mAP@0.5 为 0.869，电动车类别 mAP@0.5 为 0.952。阈值敏感性后验重算进一步显示，在 0.20--0.35 置信度阈值范围内，总体 F1 保持在 0.915--0.922 区间，电动车召回率保持在 0.956 以上。分段计时结果表明，50 张图像 batch 复查中的模型执行阶段均值约为 8.208 ms，代表性视频片段中的模型执行阶段均值约为 8.407 ms，说明 batch 端到端验证耗时不能等同于 OM/NPU 模型执行阶段耗时。上述结果说明，本文方法在当前验证集与代表性片段范围内表现出较稳定的双目标检测能力，并形成可复查的部署一致性评价证据。

本文的应用价值在于为电梯边缘安全视觉提供了一条可复查的落地路径。电梯轿厢场景的目标类别并不复杂，但系统必须在边缘设备上稳定运行，并且能够说明输出结果来自哪一组模型、阈值和后处理规则。本文将 YOLO 检测模型与海思/昇腾类开发板部署链路连接起来，使检测精度、阈值行为和运行状态能够在同一协议下被分析。这种思路也可为门厅、楼道、地下车库等封闭式公共空间的边缘视觉任务提供参考。

本文仍存在不足。首先，当前实证结果主要集中在板端完整验证、阈值后验重算和小样本分段计时，训练端、ONNX 端、OM 端和板端之间的逐样本统一匹配仍需进一步完善。其次，batch 图像验证链路端到端耗时仍然较高，完整 720 张验证集尚未使用新增计时埋点重跑分段统计，因此当前分段计时主要用于解释运行时口径，而不是替代完整验证集结果。最后，本文聚焦于行人与电动车双目标检测，尚未将人数统计、跌倒检测或其他异常行为识别纳入主体实验。后续研究可以在保持接口契约和一致性评价框架的基础上，继续扩展跨阶段样本级对齐实验、完整验证集分段计时、轻量化推理优化和更多电梯安全视觉任务。
