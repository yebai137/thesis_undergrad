\chapter{绪论}

\section{研究背景}

电梯轿厢是城市建筑中最典型的高频公共微空间之一，其空间尺度小、人员流动快、目标遮挡强、光照变化复杂，并且往往伴随镜面反光、金属门框高亮和近距离透视畸变等问题。对于通用视觉模型而言，这类成像环境与开阔道路、固定走廊或通用开放数据集存在明显差异。与此同时，电梯属于安全敏感场景，系统不仅需要“看得见”，更要“看得准、报得稳、跑得动”。一旦模型在真实部署中出现持续漏检、误检或跨端行为漂移，就可能直接影响风险治理效果。

在现实应用层面，电动车入梯是当前电梯安全治理中的高关注事件之一。与普通静态物体检测任务不同，电动车入梯往往伴随人与车同时出现、局部遮挡、仅露出车头或车把、人员身体压住车身、门口区域边框干扰等复杂情况。这意味着系统不仅要识别 \textit{ebike}，还必须同时识别 \textit{person}，并利用二者的相对关系理解事件上下文。因此，围绕人和电动车的双目标检测建立统一的边缘视觉方案，是比单类检测更贴近真实部署需求的技术路径。

随着边缘计算理论和端侧智能硬件的发展，在数据源附近完成推理与初步决策已经成为安全视觉系统的重要趋势\cite{shi2016edge}。将模型部署在边缘侧有三个直接优势：第一，原始视频不必持续上传到云端，可以降低带宽与隐私压力；第二，事件响应链路更短，更适合时延敏感的电梯预警；第三，设备运行与模型行为可以在本地形成更加稳定的评价记录。然而，边缘部署并非简单的模型格式转换，而是需要同时解决模型轻量化、量化与编译、后处理阈值稳定性、跨格式一致性等问题。

在此背景下，本文以电梯轿厢行人与电动车双目标检测为中心，研究基于 YOLO 的自训练检测模型如何兼容迁移到海思/昇腾类边缘开发板，并在设备端保持类别、边框、阈值和运行时行为的一致性。

\section{场景约束与问题挑战}

与普通目标检测相比，电梯轿厢边缘安全视觉存在更强的约束耦合。本文将这些约束概括为以下五类。

\begin{enumerate}
    \item \textbf{视角与尺度约束。} 电梯摄像机通常安装在轿厢顶部或角落，拍摄距离近、视角固定但透视变化明显。人在近景和门口区域会出现边框被截断、下半身出框或头肩放大的现象，电动车则常以局部结构出现。
    \item \textbf{反光与遮挡约束。} 金属门板、镜面装饰和玻璃表面会产生高亮或虚像，而人与电动车共同出现时又会互相遮挡，导致检测器在类别判断和边框定位上更加不稳定。
    \item \textbf{边缘部署约束。} 边缘设备通常受限于算力、功耗、存储和算子支持能力，这决定了模型不能无限增大，训练端结构也不一定能无损迁移到部署端。
    \item \textbf{运行时稳定性约束。} 安全场景更关心连续视频中的稳定表现，而不是孤立图像上的最好指标。即使某一帧检测正确，只要系统在连续数秒内频繁闪烁或类别来回跳变，也会影响真实使用。
    \item \textbf{结论可复查约束。} 论文中的实验结论不能只停留在经验判断，还需要能够回溯到验证集、指标摘要和设备端运行结果，形成从模型到系统的可复查评价依据。
\end{enumerate}

\section{国内外研究现状}

\subsection{边缘智能与轻量部署方向}

边缘计算研究已经系统性指出，在靠近数据源的位置完成计算与服务提供，是降低时延、缓解带宽压力和提升隐私保护能力的有效方式\cite{shi2016edge}。这一思想为电梯场景提供了天然契合点：视频流敏感且实时性强，如果完全依赖云端处理，不仅会产生隐私问题，也会带来额外的网络波动风险。为了让视觉模型真正适应边缘设备，研究者提出了量化、剪枝、蒸馏等模型压缩与加速方法。Jacob 等提出的整数推理友好量化框架表明，神经网络可以在低位宽下保持较好的性能\cite{jacob2018quantization}；Han 等提出的 Deep Compression 则从剪枝、量化和编码三方面给出了经典压缩思路\cite{han2016deep}；知识蒸馏方法进一步提供了从大模型向轻量模型迁移判别能力的通用工具\cite{hinton2015distilling}。

然而，边缘部署研究在实际落地时还面临一个常被忽视的问题，即兼容迁移与行为一致性。训练端的浮点模型在导出、量化和编译后，可能在边框位置、类别分数排序和阈值敏感性上出现漂移。对于普通离线基准，这种问题可能只表现为小幅指标变化；但在安全场景中，它可能被放大为连续视频中的误报、闪烁和漏检。因此，部署研究不能只停留在编译成功，还应关注跨格式链路的一致性分析\cite{stacker2021deployment,chen2021aqd}。

\subsection{轻量目标检测与实时视觉方向}

实时目标检测是当前工业视觉中最成熟的方向之一。YOLO 系列以端到端、结构简洁和工程可用性强著称，在大量真实场景中获得了广泛应用\cite{redmon2017yolo9000}。为了适配不同计算预算，研究者进一步提出了可伸缩的检测网络设计，例如 EfficientDet 通过跨尺度特征融合和复合缩放策略，在精度与计算量之间取得了较好的平衡\cite{tan2020efficientdet}；MobileNetV3 和 EfficientNet 则分别从移动端神经架构搜索和统一缩放规律角度，为轻量骨干网络设计提供了重要参考\cite{howard2019mobilenetv3,tan2019efficientnet}。此外，EdgeYOLO 等工作进一步把边缘实时检测作为设计目标，强调算子友好性、端侧吞吐和推理效率\cite{zhou2023edgeyolo}。

电梯场景中的双目标检测与通用检测任务相比更强调轻量性和稳定性。一方面，模型需要识别行人与电动车两个类别，类别数较少，理论上适合轻量模型；另一方面，轿厢内遮挡、反光和近距离尺度变化又会放大轻量模型的表达不足。因此，本文采用轻量 YOLO 模型作为实现基础，并把可部署性和一致性评价纳入方法设计，而不把模型规模对比作为当前已完成的主体实验。

\subsection{困难样本分析与数据增强方向}

当目标检测任务受到长尾样本和困难样本主导时，仅靠扩大数据量并不能稳定提升系统表现。OHEM 等在线难例挖掘方法强调，让模型更多关注当前最难的样本，可以提升训练效率并改善关键失败模式\cite{shrivastava2016ohem}。在目标检测场景中，定向数据增强也是提升泛化能力的重要手段。Scale-Aware Automatic Augmentation 等研究证明，针对目标尺度、位置和场景结构设计增强策略，往往比无差别增强更有效\cite{chen2021scaleaware}。

电梯轿厢中的困难样本具有明显场景规律：反光区域集中在门板和镜面结构，遮挡多发生在人车重叠区域，电动车局部可见常出现在门口或边缘位置。因此，本文将困难样本分类作为场景风险建模和误差解释工具，用于说明指标变化与实际轿厢结构之间的关系。针对困难样本的重加权训练、定向增强和模型尺度对比仍属于后续可扩展实验，本文不将其作为已经完成的算法改进结论。

\subsection{相关场景研究与当前不足}

与通用监控或道路交通相比，直接面向电梯视觉的公开研究数量并不多。已有工作多聚焦于人数统计、乘客计数或局部行为识别，例如基于 MobileNet-SSD 的电梯人数统计系统研究表明，在电梯场景中采用边缘侧轻量模型是具备现实可行性的\cite{lin2023elevatorcounting}。另一些研究则尝试将轻量模型用于电梯跌倒检测，说明电梯作为独立应用场景具有实际研究价值。

不过，现有研究通常只覆盖单一任务或单一部署环节，较少同时讨论电梯轿厢中的行人与电动车双目标检测、场景风险解释以及训练端到边缘端的一致性问题。本文正是在这一不足上展开：以双目标检测为主线，以场景风险建模和部署一致性评价为两条支撑线，形成面向电梯安全视觉的研究框架。


\section{问题定义与核心思路}

从系统角度看，电梯边缘安全视觉并不是单一的图像分类或离线检测任务，而是一个跨越数据、模型、编译工具和边缘设备的完整链路问题。若只在服务器端验证模型精度，无法说明模型在开发板上是否仍然保持相同输出；若只说明模型可以在板端运行，又无法解释输出结果是否与训练端语义一致。因此，本文将研究问题定义为：如何使基于 YOLO 的电梯双目标检测模型在海思/昇腾类边缘开发板上完成兼容迁移，并通过可量化指标评价部署后行为。

围绕这一问题，本文采用“问题链—技术缺口—核心处理方式”的组织思路。首先，电梯安全视觉任务需要刻画人车共现、遮挡反光和局部可见等轿厢因素，因此本文将电动车入梯治理抽象为 \textit{person + ebike} 双目标检测问题。其次，模型边缘迁移不能只被视为环境配置，输入、类别、输出和后处理约束都需要显式检查，因此本文将 PyTorch、ONNX、OM 和板端后处理抽象为迁移接口契约。最后，实验结果解释不能只依赖离线 mAP 或单一平均耗时，而需要结合完整验证集、分类别指标、阈值重算、分段计时和视频片段观察来说明部署后的模型行为。

本文的核心思路是：对于电梯轿厢这类封闭式安全场景，模型的应用价值不只取决于离线检测精度，还取决于任务定义是否贴合风险、迁移接口是否保持语义一致、实验指标是否能够解释板端真实运行口径。换言之，本文把 YOLO 双目标检测、边缘兼容迁移和部署一致性评价作为同一条研究主线，而不是三个彼此割裂的工程步骤。

该问题具有三方面研究动机。第一，电梯安全视觉对漏检和误检都敏感，模型迁移过程中的小幅数值变化可能被放大为报警行为差异。第二，YOLO 检测器虽然部署友好，但其输出张量、类别顺序和后处理参数必须被正确解释，否则板端结果会偏离训练端预期。第三，海思/昇腾类边缘平台采用专用 NPU 和离线模型格式，部署链路中存在 ONNX 导出、ATC 编译、OM 模型执行和应用层后处理等多个可能引入差异的环节。

基于上述动机，本文将“检测精度”和“部署一致性”同时作为评价目标。检测精度回答模型能否识别行人与电动车，部署一致性回答模型迁移到边缘设备后是否仍以相同语义、稳定阈值和清晰运行口径输出结果。二者结合，才能较完整地评价电梯边缘安全视觉系统的可用性。

\section{研究目标与主要内容}

本文的研究目标是：面向电梯轿厢行人与电动车双目标检测任务，构建一套基于 YOLO 的检测模型边缘兼容迁移方法，并通过统一实验协议验证模型在海思/昇腾类开发板上的检测精度、阈值行为和运行时口径。本文并不把难例分类本身作为独立算法创新，而是将其作为解释检测误差和部署稳定性的场景约束。围绕这一目标，本文主要开展以下工作。

\begin{enumerate}
    \item 建立电梯轿厢双目标检测任务定义，明确行人与电动车两个核心类别，使研究对象稳定服务于电动车入梯风险识别。
    \item 总结电梯场景中的典型困难样本，形成用于误差解释和实验分组的场景风险分类体系。
    \item 分析 YOLO 检测器在双目标任务中的结构适配方式，明确输入尺寸、类别顺序、检测头输出和 NMS 后处理对部署结果的影响。
    \item 构建从自训练权重、ONNX、OM 到边缘设备执行的兼容迁移链路，重点关注类别一致、边框解释一致、阈值稳定和运行状态可追溯。
    \item 在统一验证集、代表性片段和分段计时实验上进行分析，给出总体指标、分类别指标、阈值敏感性和边缘设备运行表现。
\end{enumerate}


\section{研究难点分析}

本文研究难点主要体现在三个层面。第一是任务层面的难点。电梯轿厢内行人与电动车经常同时出现，且存在遮挡、反光、局部可见和尺度变化等问题，模型需要在较小类别集合中保持稳定检测，而不是简单识别静态物体。

第二是模型层面的难点。YOLO 检测器具有较好的实时性，但其检测头输出、坐标解码、置信度筛选和 NMS 后处理共同决定最终结果。训练端脚本通常自动完成这些步骤，而板端应用程序需要显式实现或调用相应逻辑。只要其中一个环节解释不一致，最终输出就可能变化。

第三是部署层面的难点。海思/昇腾类边缘开发板采用专用编译和运行时体系，模型需要从训练权重转换为可执行离线模型。转换过程中的算子适配、预处理配置、数据类型变化和输出读取方式，都会影响设备端行为。因此，本文需要把部署链路纳入论文方法，而不是把它视为实验环境说明。

\section{本文的主要贡献}

本文的主要贡献可以概括为以下三点。

\begin{enumerate}
    \item \textbf{面向电梯轿厢的双目标检测任务建模。} 本文将电动车入梯治理抽象为行人与电动车双目标检测问题，并用场景风险分类解释遮挡、反光和局部可见等因素对检测输出的影响。
    \item \textbf{自训练 YOLO 模型的边缘兼容迁移方法。} 本文围绕输入预处理、类别顺序、输出张量、后处理参数和运行时约束建立迁移接口契约，使训练权重、ONNX 中间表示、OM 离线模型和板端应用之间的关键约束能够被逐项检查。
    \item \textbf{面向安全视觉的部署一致性评价。} 本文不只报告检测精度，还从类别语义、边框解释、阈值行为和运行时可靠四个角度分析模型迁移后的板端行为，为电梯边缘视觉系统提供可复查的评价方式。
\end{enumerate}

三项贡献分别对应后续章节的不同证据落点。第一项任务建模主要由第一章和第四章支撑，证据来自场景约束和验证集任务定义；第二项迁移接口契约主要在第三章展开，覆盖输入、类别、输出、后处理和运行五类约束；第三项部署一致性评价由第四章协议和第五章结果共同支撑，证据来自板端完整验证、分类别统计、阈值敏感性、NMS 敏感性、后处理消融、分段计时和视频片段分析。困难样本重加权、定向数据增强或模型尺度对比等内容尚未形成完整实测结果，本文将其放在后续可扩展方向中讨论。


\section{研究内容组织逻辑}

本文将研究重点放在“模型检测能力”和“边缘部署行为”两个层面。前者说明 YOLO 模型能否在验证集上识别行人与电动车，后者说明模型经过中间表示和离线模型格式进入边缘开发板后，能否继续保持可解释、可复查的输出。

基于这一思路，第二章介绍 YOLO 和海思/昇腾边缘部署基础，第三章集中给出兼容迁移方法，第四章定义数据、场景风险和评价协议，第五章基于已有验证结果分析设备端表现。这样的结构使模型训练、格式转换、设备端运行和实验评价形成连续链路。

\section{论文结构安排}

本文正文共分为五章。第一章介绍研究背景、场景挑战、研究目标和论文结构。第二章回顾目标检测、YOLO 方法、评价指标和海思/昇腾类边缘部署基础。第三章提出 YOLO 双目标检测模型的边缘兼容迁移方法。第四章给出数据组织、场景风险和评测协议。第五章给出实验结果与分析，最后在结论部分总结全文工作并讨论扩展方向。
